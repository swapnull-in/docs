## Overview

US sales tax is jurisdiction-level: the rate depends on the buyer's state,
county, city, and special districts. Accurate rates therefore come from an
external provider ‚Äî Recurso integrates with **TaxJar**.

The US engine applies when the **seller** is US-based (`COMPANY_COUNTRY=US`)
and the customer's billing country is `US` (or unset). Buyers outside the US
are invoiced with no US sales tax (`tax_type: "export"`).

<Info>
Without TaxJar configured, Recurso does **not** guess: US invoices carry 0%
tax, explicitly marked `sales_tax_stub`. There are no silently wrong rates.
</Info>

## TaxJar Setup

Set the API key and (optionally) the API base URL:

```bash .env
# Seller jurisdiction used by the tax engine
COMPANY_COUNTRY=US
COMPANY_STATE=CA

# TaxJar ‚Äî real US sales-tax rates. Leave unset to keep the 0% stub.
TAXJAR_API_KEY=your_taxjar_api_token
# Sandbox for testing; the default is https://api.taxjar.com
# TAXJAR_API_URL=https://api.sandbox.taxjar.com
```

On startup the API logs which mode it is in:

```text
US sales tax: TaxJar provider enabled
# or
US sales tax: 0% stub (TAXJAR_API_KEY not set)
```

At invoice time the engine sends the seller's state and the buyer's billing
`state` + `zip` to TaxJar's tax-for-order endpoint and applies the combined
jurisdiction rate. Make sure your US customers have billing addresses with
at least `state` and `zip` populated.

### Rate caching

Lookups are cached in-memory for **24 hours per buyer `(state, zip)`** pair.
Sales-tax rates change rarely, so this keeps TaxJar API spend proportional
to the number of distinct buyer locations ‚Äî not the number of invoices. The
cache lives for the process lifetime; a restart clears it.

### Error handling

- TaxJar calls time out after 10 seconds and are retried **once**, and only
  on 5xx/network failures ‚Äî auth errors (bad API key) and request-shape
  errors are never retried.
- A provider failure **never blocks invoice generation** (see the matrix
  below).

## Behavior Matrix

Every US invoice records how its tax was computed in `tax_type`:

| `tax_type` | When | Rate applied | What it means |
|------------|------|--------------|----------------|
| `sales_tax` | TaxJar configured, lookup succeeded | Live jurisdiction rate | The note names the provider and jurisdiction (e.g. `US sales tax via taxjar (CA)`). If TaxJar reports you have no nexus in the buyer's state, tax is 0 and the note flags it honestly: `no nexus in buyer state per provider ‚Äî nothing to collect`. |
| `no_nexus` | Buyer is in a US state you have **not** declared nexus in (and you've declared at least one) | 0% | The **native** nexus gate: `No sales-tax nexus in <ST> ‚Äî not collected`. Not an error ‚Äî you're correctly not collecting where you have no obligation. See [Nexus Configuration](#nexus-configuration). |
| `sales_tax_error` | TaxJar configured, lookup **failed** after the retry | 0% | The invoice ships anyway ‚Äî a tax lookup outage must never block billing. The note says `invoiced at 0% (needs review)` and a warning is logged. **Treat these invoices as an audit queue**: find them, recompute, and issue corrections. |
| `sales_tax_stub` | `TAXJAR_API_KEY` not set | 0% | The historical placeholder behavior, honestly labeled. Fine for development; not for production US billing. |
| `export` | Buyer's billing country is outside the US | ‚Äî | No US sales tax on the sale. |

<Warning>
`sales_tax_error` is deliberate honesty rather than a failure mode to hide:
Recurso chooses "invoice now at 0%, flag for review" over "block the
billing run". Monitor for it (the API logs a warning per occurrence) and
review flagged invoices promptly.
</Warning>

## Nexus Configuration

Recurso models nexus **natively**: you declare the US states where you're
registered to collect sales tax, and Recurso collects **only** there. A sale to
a buyer in a state you haven't declared is invoiced at 0% with an auditable note
‚Äî `No sales-tax nexus in <ST> ‚Äî not collected` (`tax_type: no_nexus`).

Manage your nexus states with the settings endpoints (owner/admin for dashboard
users; API keys always pass):

```bash
# List your declared nexus states
GET /v1/settings/tax/nexus

# Replace the full set (this is a full replacement, not a merge)
PUT /v1/settings/tax/nexus
{
  "states": [
    { "state_code": "CA", "nexus_type": "physical" },
    { "state_code": "NY", "nexus_type": "voluntary" }
  ]
}
```

`nexus_type` is `physical` (office / employees / inventory), `voluntary`
(registered by choice), or `economic` (a threshold was crossed ‚Äî see below);
it defaults to `physical`.

### How the native gate interacts with TaxJar

Native nexus and the provider are **both** gates ‚Äî a sale is taxed only where
**both** allow it:

- **You've declared at least one nexus state:** Recurso collects only in your
  declared states; buyers elsewhere get `no_nexus` (0%). Where you *do* have
  nexus, TaxJar (if configured) computes the exact rate ‚Äî and can still
  independently return 0% for a specific exempt address (`no nexus in buyer
  state per provider`).
- **You've declared no nexus states:** the native gate is **off** for backward
  compatibility ‚Äî tax falls through to TaxJar's own nexus configuration (or the
  0% `sales_tax_stub` if TaxJar isn't configured). This is the pre-nexus
  behavior.

So a declared state is *necessary* to collect, and the provider can still exempt
a specific address ‚Äî Recurso never collects where you haven't declared nexus.

<Info>
Beyond manually declared `physical` / `voluntary` nexus, Recurso **tracks
economic-nexus thresholds** ($100k / 200 transactions, varying by state) from
your year-to-date sales and transaction counts, and auto-establishes `economic`
nexus when you cross one. Monitor per-state proximity with
`GET /v1/settings/tax/nexus/status` ‚Äî see the
[US sales tax setup tutorial](/tutorials/us-sales-tax-setup) and the
[nexus status endpoint](/api-reference/settings/nexus-status).
</Info>

<Warning>
The seeded threshold dataset self-reports `dataset_certified: false` until it
passes professional review. While it's `false`, treat threshold crossings as
**indicative** ‚Äî confirm each state's current rule with a tax professional before
relying on it for a filing or registration decision.
</Warning>

## Testing

Point `TAXJAR_API_URL` at TaxJar's sandbox to test the full path without
live rates:

```bash .env
TAXJAR_API_KEY=your_sandbox_token
TAXJAR_API_URL=https://api.sandbox.taxjar.com
```

Then create a US customer with a full billing address and generate an
invoice ‚Äî the invoice's `tax_type` should be `sales_tax`.
