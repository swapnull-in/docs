## Installation

<Tip>
**Fastest path:** clone the repository and run `make demo`. It builds and starts the full stack (API, dashboard, PostgreSQL, TigerBeetle, Mailhog), waits for the API to become healthy, and loads demo data. Then open the dashboard at `http://localhost:5173` and log in with API key `sk_test_12345`.
</Tip>

Choose your preferred setup method:

<Tabs>
  <Tab title="Docker (Recommended)">
    ```bash
    # Clone the repository
    git clone https://github.com/recurso-dev/recurso.git
    cd recur-so

    # Start with Docker Compose
    docker-compose up -d

    # API available at http://localhost:8080
    ```
  </Tab>
  <Tab title="From Source">
    ```bash
    # Clone the repository
    git clone https://github.com/recurso-dev/recurso.git
    cd recur-so

    # Install dependencies
    go mod download

    # Set up environment
    cp .env.example .env

    # Run migrations
    make migrate

    # Start the server
    make run
    ```
  </Tab>
  <Tab title="Cloud (Early Access)">
    Recurso Cloud is managed hosting — no installation required.

    1. **Join the waitlist** — email [cloud@recurso.dev](mailto:cloud@recurso.dev) to request access
    2. **Receive your API credentials** — we'll provision your instance and send you an API key
    3. **Test the connection:**

    ```bash
    curl https://your-instance.recurso.cloud/v1/customers \
      -H "Authorization: Bearer $RECURSO_API_KEY"
    ```

    The rest of this quickstart (creating plans, customers, and subscriptions) works identically on Cloud. Replace `http://localhost:8080` with your Cloud instance URL.

    [Compare plans](/pricing)
  </Tab>
</Tabs>

## Configuration

Set these environment variables:

```bash .env
# Database
DATABASE_URL=postgres://user:pass@localhost:5432/recurso

# API
PORT=8080
API_KEY_SECRET=your-secret-key

# Payment Gateways (optional)
RAZORPAY_KEY_ID=rzp_test_xxx
RAZORPAY_KEY_SECRET=xxx
STRIPE_SECRET_KEY=sk_test_xxx

# Email (optional)
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USER=noreply@example.com
SMTP_PASSWORD=xxx
```

## Create an API Key

1. Start the dashboard — a React app in the `frontend/` directory of the repository:

   ```bash
   cd frontend
   npm install
   npm run dev
   ```

2. Open the dashboard at `http://localhost:5173` (the Vite dev server proxies API calls to the API at `http://localhost:8080`)
3. Navigate to **Settings → API Keys**
4. Click **Create API Key**
5. Copy and store securely

Test the connection:

```bash
curl http://localhost:8080/v1/customers \
  -H "Authorization: Bearer your-api-key"
```

## Optional: Load Demo Data

Want a populated dashboard before creating anything yourself? Seed the dev
database with a sample tenant — plans, customers, subscriptions, invoices,
events, and a webhook endpoint:

```bash
make seed
```

<Warning>
`make seed` wipes all existing data in the target database before loading
the demo dataset. Only use it on a fresh local install.
</Warning>

Then log in to the dashboard with the demo API key `sk_test_12345` and
explore. When you're done exploring, run `make seed` again for a clean slate
or register your own tenant.

In dev, any emails the system sends (invoices, dunning notices, and so on)
are captured by Mailhog — view them at `http://localhost:8025`.

## Create Your First Plan

<Steps>
  <Step title="Create a pricing plan">
    ```bash
    curl -X POST http://localhost:8080/v1/plans \
      -H "Authorization: Bearer your-api-key" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Pro Plan",
        "description": "For growing teams",
        "interval": "month",
        "prices": [
          { "amount": 4999, "currency": "INR" },
          { "amount": 999, "currency": "USD" }
        ]
      }'
    ```
  </Step>

  <Step title="Add a customer">
    ```bash
    curl -X POST http://localhost:8080/v1/customers \
      -H "Authorization: Bearer your-api-key" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Acme Corp",
        "email": "billing@acme.com",
        "metadata": { "company_size": "mid-market" }
      }'
    ```
  </Step>

  <Step title="Start a subscription">
    ```bash
    curl -X POST http://localhost:8080/v1/subscriptions \
      -H "Authorization: Bearer your-api-key" \
      -H "Content-Type: application/json" \
      -d '{
        "customer_id": "cust_abc",
        "plan_id": "plan_pro",
        "payment_gateway": "razorpay"
      }'
    ```
  </Step>

  <Step title="Handle webhooks">
    ```javascript
    import express from 'express';
    import crypto from 'crypto';

    const app = express();

    app.post('/webhooks/recurso', express.json(), (req, res) => {
      const signature = req.headers['recurso-signature'];
      const expected = crypto
        .createHmac('sha256', process.env.WEBHOOK_SECRET)
        .update(JSON.stringify(req.body))
        .digest('hex');

      if (signature !== expected) {
        return res.status(401).json({ error: 'Invalid signature' });
      }

      switch (req.body.type) {
        case 'subscription.activated':
          // Grant access to your product
          break;
        case 'payment.succeeded':
          // Payment received
          break;
        case 'subscription.cancelled':
          // Revoke access
          break;
      }

      res.sendStatus(200);
    });
    ```
  </Step>
</Steps>
