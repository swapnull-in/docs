Recurso ships three official SDKs — **Node.js**, **Python**, and **Go** — each a
typed, resource-oriented client over the REST API covering the full surface:
plans, customers, the subscription lifecycle, invoices, the usage platform
(recording, windowed queries, per-period reports), quotes, entitlements,
webhooks (with delivery tracking and redelivery), analytics, and more.

| Language | Package | Install |
| --- | --- | --- |
| Node.js / TypeScript | `recurso-node` | from `sdk/node` (see below) |
| Python 3.11+ | `recurso` | `pip install ./sdk/python` |
| Go 1.22+ | `github.com/recurso-dev/recurso-go` | `go get github.com/recurso-dev/recurso-go` |

<Info>
The Node and Python packages aren't published to npm / PyPI yet — install them
from the [main repository](https://github.com/recurso-dev/recurso/tree/main/sdk)
for now. The Go module is `go get`-able directly from its import path.
</Info>

All three authenticate the same way — your API key is sent as a bearer token
(`Authorization: Bearer <key>`) — and mirror the same resources and method names.
The worked examples in **Node.js** below are the fullest reference; the
[Python](#python) and [Go](#go) quickstarts follow at the end.

## Node.js

The Node SDK (`recurso-node`) is the fullest reference — 17 resources and 67
methods. Install it from the repo:

## Installation

```bash
# From a checkout of the recurso repository
cd sdk/node
npm install && npm run build
npm link          # or: npm pack and install the tarball in your project
```

## Setup

```typescript
import { Recurso } from 'recurso-node';

const recurso = new Recurso(
  'rsk_test_your_api_key',
  'http://localhost:8080'   // your API base URL
);
```

## Quick example

```typescript
// Create a plan
const plan = await recurso.plans.create({
  name: 'Pro Plan',
  code: 'PRO-USD',
  amount: 2900,            // minor units (cents/paise)
  currency: 'USD',
  interval_unit: 'month',
});

// Create a customer
const customer = await recurso.customers.create({
  name: 'Jane User',
  email: 'jane@example.com',
  country: 'US',
});

// Subscribe them
const subscription = await recurso.subscriptions.create({
  customer_id: customer.id,
  plan_id: plan.id,
});
```

## Common flows

### Subscription lifecycle

```typescript
// Cancel at period end
await recurso.subscriptions.cancel(sub.id, { cancel_at_period_end: true });

// Pause and resume
await recurso.subscriptions.pause(sub.id);
await recurso.subscriptions.resume(sub.id);

// Bill 3 future periods immediately (advance invoicing)
await recurso.subscriptions.advance(sub.id, { periods: 3 });
```

### Usage-based billing

```typescript
// Record a metered event (the subscription must belong to your tenant,
// and customer_id must match the subscription's customer)
await recurso.usage.record({
  subscription_id: sub.id,
  customer_id: customer.id,
  dimension: 'api_calls',
  quantity: 1500,
});

// Time-windowed buckets for charts (defaults: last 30 days, day buckets)
const chart = await recurso.usage.query({
  subscription_id: sub.id,
  dimension: 'api_calls',
  granularity: 'day',        // 'day' | 'month'
});
// { data: [{ period, dimension, quantity }, ...], from, to, granularity }

// The tenant's dimension catalog with first/last seen and event counts
const catalog = await recurso.usage.dimensions();

// Current-period + lifetime usage per dimension, with the customer's
// entitlement limit/remaining joined in where a feature_key matches the
// dimension name — the "4,231 of 10,000 API calls" view
const usage = await recurso.subscriptions.usage(sub.id);
// { dimensions: [{ dimension, period_quantity, lifetime_quantity,
//                  limit_value, remaining }], ... }
```

See the [Usage-Based Billing guide](/advanced/usage-billing) for the
metering-plus-entitlements headroom pattern.

### Entitlements and feature gating

```typescript
// Grant features on a plan (full replacement — send the complete set)
await recurso.entitlements.setForPlan(plan.id, [
  { feature_key: 'sso', kind: 'boolean', bool_value: true },
  { feature_key: 'seats', kind: 'limit', limit_value: 25 },
]);

// Fast per-request check (union across active + trialing subscriptions)
const check = await recurso.entitlements.check(customer.id, 'sso');
if (check.granted) {
  // feature enabled
}
```

See the [Entitlements guide](/advanced/entitlements) for resolution
semantics and the full gating pattern.

### Webhook endpoints

```typescript
await recurso.webhooks.create({
  url: 'https://example.com/hooks/recurso',
  event_types: ['invoice.paid', 'subscription.canceled'],
});
```

### Delivery tracking and redelivery

```typescript
// Recent failed deliveries to one endpoint
const failing = await recurso.webhooks.deliveries(endpoint.id, {
  status: 'failed',   // pending | succeeded | failed
  limit: 20,
});

// All delivery attempts of one event, across endpoints
const attempts = await recurso.events.deliveries(event.id);

// Re-queue a failed event (202: { event_id, deliveries_queued })
await recurso.events.redeliver(event.id);
```

See the [Webhooks guide](/advanced/webhooks#delivery-tracking--redelivery)
for status derivation and retry semantics.

### FX-normalized MRR

```typescript
const metrics = await recurso.analytics.mrr();
// { mrr, normalized_mrr, reporting_currency, breakdown: [...],
//   fx: { rates, source: 'live' | 'static-fallback', as_of } }
```

### Invoice PDFs

```typescript
const invoices = await recurso.invoices.list({ status: 'paid' });
const url = recurso.invoices.pdfUrl(invoices.data[0].id);
```

## Python

The Python SDK (`recurso`) is generated from the OpenAPI 3.1 spec, so it tracks
the REST surface exactly, with typed request/response models and both **sync and
async** call styles. Requires Python 3.11+.

```bash
pip install ./sdk/python          # from the repo root
# or, editable, for development:
pip install -e ./sdk/python
```

```python
from recurso import AuthenticatedClient
from recurso.api.plans import create_plan
from recurso.api.customers import create_customer
from recurso.api.subscriptions import create_subscription
from recurso.api.entitlements import check_entitlement
from recurso.models import (
    CreatePlanRequest,
    CreatePlanRequestIntervalUnit,
    CreateCustomerRequest,
    CreateSubscriptionRequest,
)

client = AuthenticatedClient(
    base_url="https://billing.example.com",
    token="rsk_test_your_api_key",   # sent as: Authorization: Bearer <token>
)

with client:
    plan = create_plan.sync(
        client=client,
        body=CreatePlanRequest(
            name="Pro Plan",
            code="PRO-USD",
            amount=2900,  # minor units (cents/paise)
            currency="USD",
            interval_unit=CreatePlanRequestIntervalUnit.MONTH,
            interval_count=1,
        ),
    )

    customer = create_customer.sync(
        client=client,
        body=CreateCustomerRequest(
            name="Jane User",
            email="jane@example.com",
            country="US",
        ),
    )

    subscription = create_subscription.sync(
        client=client,
        body=CreateSubscriptionRequest(customer_id=customer.id, plan_id=plan.id),
    )

    # Fast single-feature entitlement check (the feature-gating hot path)
    result = check_entitlement.sync(client=client, customer_id=customer.id, feature="sso")
```

Every endpoint module exposes four functions: `sync` and `sync_detailed`
(blocking), and `asyncio` / `asyncio_detailed` (awaitable). Mutating endpoints
support idempotency via a per-call header:

```python
client = client.with_headers({"X-Idempotency-Key": "order-1234"})
```

## Go

The Go SDK (`recurso-go`) is hand-crafted from the same OpenAPI spec — 18
resources / 68 methods, **standard library only** (no third-party deps).
Requires Go 1.22+. Monetary amounts are `int64` in the currency's smallest unit.

```bash
go get github.com/recurso-dev/recurso-go
```

```go
package main

import (
	"context"
	"errors"
	"log"

	recurso "github.com/recurso-dev/recurso-go"
)

func main() {
	ctx := context.Background()
	client := recurso.NewClient(
		"rsk_test_your_api_key",
		recurso.WithBaseURL("https://billing.example.com/v1"), // default: https://api.recurso.dev/v1
	)

	plan, err := client.Plans.Create(ctx, &recurso.PlanCreateParams{
		Name:          "Pro Plan",
		Code:          "PRO-USD",
		Amount:        2900, // minor units ($29.00)
		Currency:      "USD",
		IntervalUnit:  "month",
		IntervalCount: 1,
	})
	if err != nil {
		log.Fatal(err)
	}

	customer, err := client.Customers.Create(ctx, &recurso.CustomerCreateParams{
		Name:    "Jane User",
		Email:   "jane@example.com",
		Country: "US",
	})
	if err != nil {
		log.Fatal(err)
	}

	sub, err := client.Subscriptions.Create(ctx, &recurso.SubscriptionCreateParams{
		CustomerID: customer.ID,
		PlanID:     plan.ID,
	})
	if err != nil {
		// Any non-2xx response decodes to *recurso.APIError.
		var apiErr *recurso.APIError
		if errors.As(err, &apiErr) {
			log.Fatalf("recurso: %s (%s, HTTP %d)", apiErr.Message, apiErr.Code, apiErr.StatusCode)
		}
		log.Fatal(err)
	}

	log.Printf("created subscription %s (%s)", sub.ID, sub.Status)
}
```

Every method takes a `context.Context` first and returns `(T, error)`;
non-2xx responses come back as `*recurso.APIError` carrying the standard
`{"error": {"code", "message"}}` envelope and the HTTP status.

## Other languages

Node.js, Python, and Go are the official SDKs. For any other language, call the
[REST API](/api-reference/introduction) directly — every endpoint is documented
with `curl` examples, and all three SDKs are generated from the same
[OpenAPI 3.1 spec](/api-reference/introduction) you can point your own generator at.
