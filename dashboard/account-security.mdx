## Overview

The Recurso dashboard signs in with **per-user accounts** (email + password), scoped
to your workspace. This page covers the self-service security features every user
has: resetting a password, enabling **two-factor authentication (2FA)**, and managing
**active sessions**.

<Note>
  Dashboard login is separate from **API keys**. API keys authenticate machine-to-machine
  requests and are managed under **Developers** — see
  [API Authentication](/api-reference/authentication). Your login credentials never
  grant API access, and vice-versa.
</Note>

## Resetting your password

<Steps>
  <Step title="Start the reset">
    On the login screen, click **Forgot password?** and enter your email.
  </Step>
  <Step title="Check your email">
    If an account exists for that address, Recurso emails a reset link. The link is
    single-use and expires after **1 hour**.
  </Step>
  <Step title="Choose a new password">
    Open the link and set a new password (minimum 8 characters).
  </Step>
  <Step title="Sign back in">
    All of your existing sessions are signed out automatically, so anyone using a stolen
    session is kicked out. Log in again with the new password.
  </Step>
</Steps>

<Note>
  For security, the "check your email" confirmation is shown **whether or not** the
  address matches an account — this prevents attackers from discovering which emails
  are registered.
</Note>

<Info>
  **Self-hosting:** reset emails are sent through your configured SMTP server, and the
  reset link points at `${DASHBOARD_URL}/reset-password`. Set `DASHBOARD_URL` so links
  resolve to your dashboard's public address.
</Info>

## Two-factor authentication (2FA)

Recurso supports **TOTP** (time-based one-time codes) from any standard authenticator
app — Google Authenticator, 1Password, Authy, and others.

### Enable 2FA

<Steps>
  <Step title="Open Security">
    In the dashboard sidebar, go to **Security**.
  </Step>
  <Step title="Start setup">
    Click **Enable two-factor authentication**.
  </Step>
  <Step title="Scan the QR code">
    Scan the QR code with your authenticator app, or enter the shown key manually. The
    secret is generated in your browser and is never sent to a third party.
  </Step>
  <Step title="Confirm">
    Enter the 6-digit code from the app to verify and turn on 2FA.
  </Step>
  <Step title="Save your backup codes">
    Recurso shows **10 one-time backup codes**. Save them somewhere safe — each can be
    used once if you lose access to your authenticator, and they are **not shown again**.
  </Step>
</Steps>

### Signing in with 2FA

Once 2FA is on, login is two steps: after your email and password, Recurso asks for a
**6-digit code**. Enter the code from your authenticator app — or one of your **backup
codes** if you don't have your device (each backup code works only once).

### Disable 2FA

Go to **Security → Disable two-factor authentication** and confirm with a current code
(a TOTP code or a backup code). Disabling wipes the stored secret and all backup codes.

<Warning>
  If you lose both your authenticator **and** your backup codes, an owner or admin on
  your workspace can remove your user and re-invite you. Keep at least one backup code
  in a password manager.
</Warning>

## Active sessions

Every login creates a session, stored as a secure, httpOnly cookie. Under
**Security → Active sessions** you can see each of your logins — the device
(browser/user-agent) and when it started — with the current one marked **This device**.

- **Revoke** any individual session to sign that device out.
- **Log out all other sessions** ends every login except the one you're using — useful
  after a lost laptop or a shared computer.

<Note>
  Sessions expire automatically after `SESSION_TTL_HOURS` (default **7 days**). Changing
  your password revokes all sessions immediately.
</Note>
