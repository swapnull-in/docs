## Overview

Plans define what you're selling and how much it costs. Each plan has:

- **Name & Description** - What appears to customers
- **Interval** - Billing frequency (monthly, yearly, etc.)
- **Prices** - Amount in one or more currencies
- **Features** - What's included (for display)

## Create a Plan

<CodeGroup>
```typescript TypeScript
const plan = await recurso.plans.create({
  name: 'Pro Plan',
  description: 'For growing teams',
  interval: 'month',
  trial_days: 14,
  prices: [
    { amount: 4999, currency: 'INR' },  // ‚Çπ49.99
    { amount: 999, currency: 'USD' }    // $9.99
  ],
  features: [
    'Up to 10 team members',
    'Unlimited projects',
    'Priority support'
  ]
});
```

```bash cURL
curl -X POST https://api.recurso.dev/v1/plans \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Pro Plan",
    "description": "For growing teams",
    "interval": "month",
    "trial_days": 14,
    "prices": [
      { "amount": 4999, "currency": "INR" },
      { "amount": 999, "currency": "USD" }
    ]
  }'
```
</CodeGroup>

## Pricing Models

A plan always carries a **flat recurring price** (`amount` + `currency`).
Everything else ‚Äî seats, tiers, metered usage ‚Äî composes on top of it.

<Tabs>
  <Tab title="Flat rate">
    One fixed price per period. This is the plan itself:

    ```bash
    curl -X POST https://api.recurso.dev/v1/plans \
      -H "Authorization: Bearer $API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "name": "Starter",
        "code": "starter-monthly",
        "interval_unit": "month",
        "interval_count": 1,
        "amount": 99900,
        "currency": "INR"
      }'
    ```

    `amount` is in minor units ‚Äî `99900` is ‚Çπ999.00.
  </Tab>
  <Tab title="Per-seat">
    Two ways to bill seats:

    **Add-ons** ‚Äî attach a per-seat plan to the subscription with a
    quantity. The add-on bills as its own line (price √ó quantity) from the
    next recurring invoice:

    ```bash
    curl -X POST https://api.recurso.dev/v1/subscriptions/{sub_id}/addons \
      -H "Authorization: Bearer $API_KEY" \
      -H "Content-Type: application/json" \
      -d '{ "plan_id": "plan_seat", "quantity": 10 }'
    ```

    **Seat metering** ‚Äî define a billable metric with `max(seats)`
    aggregation and report seat counts as usage events. The period bills
    the high-water mark; no proration bookkeeping needed. See
    [usage-based billing](/advanced/usage-billing).
  </Tab>
  <Tab title="Tiered / metered">
    Usage pricing lives in **plan charges**, not on the plan price. Attach
    charges that reference billable metrics, one of four charge models:

    ```bash
    curl -X PUT https://api.recurso.dev/v1/plans/{plan_id}/charges \
      -H "Authorization: Bearer $API_KEY" \
      -H "Content-Type: application/json" \
      -d '[{
        "metric_code": "api_calls",
        "charge_model": "graduated",
        "amounts": { "INR": { "tiers": [
          { "up_to": 100000, "unit_amount": "0" },
          { "up_to": 1000000, "unit_amount": "0.05" },
          { "up_to": null, "unit_amount": "0.0035" }
        ] } }
      }]'
    ```

    The `PUT` is a full replacement ‚Äî charges absent from the request are
    removed. Models: `per_unit`, `graduated`, `volume`, `package` ‚Äî
    [compared side by side here](/advanced/usage-billing#charge-models).
  </Tab>
  <Tab title="Hybrid">
    A plan holding **both** a flat price and charges is hybrid: the flat
    fee bills in advance (as always), usage bills in arrears on the same
    renewal invoice at period close ‚Äî each as its own tax-resolved line.

    ```text
    Pro Plan (advance)                 ‚Çπ4,999.00
    API calls ‚Äî 45,000 √ó usage         ‚Çπ1,750.00
    IGST (18%)                         ‚Çπ1,214.82
    ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ‚îÄ
    Total                              ‚Çπ7,963.82
    ```
  </Tab>
</Tabs>

## Plan Intervals

| Interval | Value | Example |
|----------|-------|---------|
| Daily | `day` | Metered services |
| Weekly | `week` | Short-term access |
| Monthly | `month` | Most common |
| Yearly | `year` | Discounted annual |

For custom intervals, use `interval_count`:

```typescript
{
  interval: 'month',
  interval_count: 3  // Quarterly billing
}
```

## Trials

Add free trial periods to any plan:

```typescript
{
  name: 'Pro Plan',
  trial_days: 14  // 14-day free trial
}
```

<Info>
Trial subscribers don't pay until the trial ends. No payment method required during trial (configurable).
</Info>

## List Plans

Retrieve plans with optional filters and pagination:

<CodeGroup>
```typescript TypeScript
const plans = await recurso.plans.list({
  active: true,
  limit: 10,
  offset: 0
});

// Returns
{
  data: [
    { id: 'plan_starter', name: 'Starter', ... },
    { id: 'plan_pro', name: 'Pro', ... }
  ],
  has_more: false
}
```

```bash cURL
curl -G https://api.recurso.dev/v1/plans \
  -H "Authorization: Bearer $API_KEY" \
  -d active=true \
  -d limit=10
```
</CodeGroup>

### Filter Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `active` | `boolean` | Filter by active/archived status |
| `interval` | `string` | Filter by billing interval (`month`, `year`, etc.) |
| `limit` | `integer` | Number of results per page (default 20, max 100) |
| `offset` | `integer` | Number of results to skip for pagination |

## Multi-Currency Pricing

Add prices in multiple currencies so Recurso auto-selects the correct price based on the customer's currency or location:

```typescript
const plan = await recurso.plans.create({
  name: 'Global Pro',
  interval: 'month',
  prices: [
    { amount: 4999, currency: 'INR' },
    { amount: 999, currency: 'USD' },
    { amount: 899, currency: 'EUR' },
    { amount: 799, currency: 'GBP' }
  ]
});
```

When a subscription is created, Recurso matches the plan price to the customer's currency. If no matching currency is found, the subscription creation returns an error.

See the [Multi-Currency guide](/advanced/multi-currency) for exchange rate handling and reporting.

## Archive a Plan

Archived plans can't be used for new subscriptions but existing subscriptions continue unaffected:

<CodeGroup>
```typescript TypeScript
await recurso.plans.archive('plan_old');
```

```bash cURL
curl -X POST https://api.recurso.dev/v1/plans/plan_old/archive \
  -H "Authorization: Bearer $API_KEY"
```
</CodeGroup>

<Warning>
Archiving is not reversible. If you need the same plan again, create a new one. Existing subscribers on the archived plan continue billing normally ‚Äî only new sign-ups are blocked.
</Warning>

## Plan Versioning Strategy

When you need to change pricing, you have two options:

| Approach | When to Use |
|----------|-------------|
| **Create a new plan** | You want existing subscribers to keep their current rate. New customers get the new price. |
| **Update the existing plan** | You want the price change to apply to all subscribers at their next renewal. |

The recommended approach is to **create a new plan** for each pricing change. This gives you a clear audit trail and avoids unexpected charge changes for existing customers.

```typescript
// v1 ‚Äî original pricing
const v1 = await recurso.plans.create({
  name: 'Pro Plan',
  interval: 'month',
  prices: [{ amount: 4999, currency: 'INR' }]
});

// v2 ‚Äî price increase, archive v1
await recurso.plans.archive(v1.id);
const v2 = await recurso.plans.create({
  name: 'Pro Plan',
  interval: 'month',
  prices: [{ amount: 5999, currency: 'INR' }]
});
// Existing v1 subscribers keep paying ‚Çπ49.99
// New subscribers sign up at ‚Çπ59.99
```
