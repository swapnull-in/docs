## Overview

Customers represent the people or companies who pay you. Each customer can have:

- Multiple subscriptions
- Multiple payment methods
- Billing details (address, tax ID)
- Custom metadata

## Create a Customer

<CodeGroup>
```typescript TypeScript
const customer = await recurso.customers.create({
  name: 'Acme Corporation',
  email: 'billing@acme.com',
  phone: '+919876543210',
  address: {
    line1: '123 MG Road',
    city: 'Bangalore',
    state: 'Karnataka',
    postal_code: '560001',
    country: 'IN'
  },
  metadata: {
    company_id: 'acme_123',
    plan_type: 'enterprise'
  }
});
```

```bash cURL
curl -X POST https://api.recurso.dev/v1/customers \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Acme Corporation",
    "email": "billing@acme.com"
  }'
```
</CodeGroup>

## B2B Customers (India)

For business customers in India, capture GSTIN for proper invoicing:

```typescript
const b2bCustomer = await recurso.customers.create({
  name: 'TechCorp India Pvt Ltd',
  email: 'accounts@techcorp.in',
  gstin: '29ABCDE1234F1Z5',  // 15-character GSTIN
  address: {
    line1: '456 Koramangala',
    city: 'Bangalore',
    state: 'Karnataka',  // Important for CGST/SGST vs IGST
    postal_code: '560034',
    country: 'IN'
  }
});
```

<Info>
When GSTIN is provided, invoices automatically show the customer's business name and tax registration number. See the [GST Invoicing guide](/compliance/gst-invoicing) for tax calculation details.
</Info>

## List and Search Customers

Retrieve customers with filters for email, country, or pagination:

<CodeGroup>
```typescript TypeScript
const customers = await recurso.customers.list({
  email: 'billing@acme.com',  // Exact match
  country: 'IN',              // Filter by country code
  limit: 20,
  offset: 0
});
```

```bash cURL
curl -G https://api.recurso.dev/v1/customers \
  -H "Authorization: Bearer $API_KEY" \
  -d email=billing@acme.com \
  -d country=IN \
  -d limit=20
```
</CodeGroup>

### Filter Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `email` | `string` | Filter by exact email address |
| `country` | `string` | Filter by ISO 3166-1 alpha-2 country code |
| `limit` | `integer` | Results per page (default 20, max 100) |
| `offset` | `integer` | Number of results to skip |

## Customer Portal

Each customer gets access to a self-service portal:

```typescript
// Generate a magic link for customer portal access
const session = await recurso.portal.createSession({
  customer_id: 'cust_abc123',
  return_url: 'https://yourapp.com/account'
});

// Redirect customer to:
console.log(session.url);
// https://portal.yourapp.com/verify?token=xxx
```

The portal allows customers to:
- View and pay invoices
- Update payment methods
- Cancel subscriptions
- Download invoices as PDF

See the [Customer Portal guide](/portal/overview) for full setup instructions.

## Payment Methods

Associate payment methods with customers:

```typescript
// After collecting payment via Razorpay/Stripe
await recurso.customers.attachPaymentMethod('cust_abc123', {
  gateway: 'razorpay',
  gateway_payment_method_id: 'method_xxx',
  set_default: true
});
```

### Update Payment Method

To update a customer's payment method (e.g., when a card expires), use the update payment method endpoint:

<CodeGroup>
```typescript TypeScript
await recurso.customers.updatePaymentMethod('cust_abc123', {
  gateway: 'razorpay',
  gateway_token: 'token_new_card',
  set_default: true
});
```

```bash cURL
curl -X PUT https://api.recurso.dev/v1/customers/cust_abc123/payment-method \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "gateway": "razorpay",
    "gateway_token": "token_new_card",
    "set_default": true
  }'
```
</CodeGroup>

<Tip>
When a payment fails due to an expired card, send the customer to the [Customer Portal](/portal/overview) to update their payment method, or use [Magic Links](/portal/magic-links) to send them a direct update link.
</Tip>

## Churn Prediction

Recurso tracks customer engagement signals and assigns a churn risk score. Use this to proactively intervene with at-risk customers:

```typescript
const score = await recurso.churn.getCustomerScore('cust_abc123');

// Returns
{
  customer_id: 'cust_abc123',
  risk_score: 0.73,  // 0-1, higher = more likely to churn
  risk_level: 'high',
  factors: [
    'Payment failed 2 times in last 30 days',
    'No login activity for 14 days'
  ]
}
```

See the [Churn Prediction guide](/advanced/churn-prediction) for setting up alerts and automated interventions.

## Delete a Customer

<Warning>
Deleting a customer cancels all active subscriptions and is irreversible. Consider archiving instead if you may need the data later.
</Warning>

```typescript
await recurso.customers.delete('cust_abc123');
```

Before deleting, check for:
- Active subscriptions (will be cancelled immediately)
- Open invoices (will be voided)
- Outstanding credit balance (will be forfeited)

## Metadata

Store custom data on customers:

```typescript
await recurso.customers.update('cust_abc123', {
  metadata: {
    salesforce_id: 'SF123456',
    segment: 'enterprise',
    account_manager: 'john@company.com'
  }
});
```

<Tip>
Use metadata to link Recurso customers to your internal systems.
</Tip>
