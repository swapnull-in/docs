---
title: "Get EU e-invoice"
openapi: "GET /v1/invoices/{id}/eu-einvoice"
description: "Retrieve an invoice's generated EN 16931 / UBL e-invoice and its delivery status."
---

## Path parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | string | Yes | The invoice ID. |

Returns the invoice's EU e-invoice record, or `data: null` when none has been
generated (the tenant hasn't opted in, or the invoice isn't EU-scoped).

## Example request

```bash
curl https://api.recurso.dev/v1/invoices/inv_Rp3mK7nBwQ/eu-einvoice \
  -H "Authorization: Bearer $API_KEY"
```

## Example response

```json
{
  "data": {
    "id": "euinv_9fA2",
    "invoice_id": "inv_Rp3mK7nBwQ",
    "syntax": "ubl21",
    "status": "sent",
    "document": "<Invoice xmlns=\"urn:oasis:names:...\">…</Invoice>",
    "message_id": "ap-msg-7c1e",
    "error_message": "",
    "created_at": "2026-07-02T10:30:00Z"
  }
}
```

<Note>
`status` is one of `generated`, `sent`, or `failed`. The `document` field holds
the full EN 16931 UBL 2.1 XML — the same bytes handed to the Access Point.
</Note>
