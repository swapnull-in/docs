---
title: "List Webhook Deliveries"
openapi: "GET /v1/webhooks/{id}/deliveries"
description: "Recent delivery attempts to a webhook endpoint"
---

Returns recent delivery attempts to this endpoint, newest first. Use it as the delivery log for a single receiver — e.g. to spot an endpoint that started failing after a deploy.

## Path Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `id` | string (uuid) | The webhook endpoint ID |

## Query Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `status` | string | Filter by derived delivery status: `pending`, `succeeded`, or `failed` |
| `limit` | integer | Max results (default: 50) |
| `offset` | integer | Pagination offset (default: 0) |

## Example Request

```bash
curl "https://api.recurso.dev/v1/webhooks/wh_abc123/deliveries?status=failed&limit=20" \
  -H "Authorization: Bearer $API_KEY"
```

## Response

```json
{
  "data": [
    {
      "id": "del_5b4a3c2d",
      "event_id": "evt_a1b2c3d4",
      "webhook_endpoint_id": "wh_abc123",
      "endpoint_url": "https://yourapp.com/webhooks/recurso",
      "status": "failed",
      "attempts": 5,
      "last_status_code": 500,
      "last_error": "HTTP 500: internal server error",
      "created_at": "2026-07-05T22:14:00Z"
    }
  ]
}
```

Delivery fields and the `pending` / `succeeded` / `failed` status derivation are the same as [List Event Deliveries](/api-reference/events/deliveries).

<Tip>
A delivery becomes `failed` after 5 attempts. To try again, redeliver the underlying event with [`POST /v1/events/{id}/redeliver`](/api-reference/events/redeliver).
</Tip>
