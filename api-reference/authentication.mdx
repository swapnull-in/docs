## API Keys

Authenticate all API requests with your secret API key in the `Authorization`
header as a Bearer token:

```bash
curl https://api.recurso.dev/v1/customers \
  -H "Authorization: Bearer $RECURSO_API_KEY"
```

Keys are secret credentials prefixed with `sk_` (for example
`sk_live_9f8e7d6c…`). The demo dataset ships with the fixed key `sk_test_12345`
for local exploration.

<Warning>
**Never expose API keys in client-side code.** Keys are secret and must only be
used from your backend.
</Warning>

## Test vs. live

Recurso issues a single kind of secret key — there is **no separate "test" API
key**. Whether requests move real money is determined by the **payment-gateway
keys configured on the server**, not by your Recurso key. The API reports the
current mode at `GET /version`:

| `gateway_mode` | Meaning |
| --- | --- |
| `none` | No gateway configured — the mock gateway simulates payments (full sandbox) |
| `test` | Stripe `sk_test…` / Razorpay `rzp_test…` keys — real gateway test mode |
| `live` | Real gateway keys — real money |

The dashboard's **Test mode** chip is driven by this same value. To build and
test safely, run against an instance in `none` or `test` mode — see
[Testing & Sandbox](/testing).

## Generate a key

1. Go to **Settings → API Keys** in your dashboard
2. Click **Create API Key**
3. Copy and store it securely — the full key is shown only once

## Usage

Pass your API key in the `Authorization` header as a Bearer token:

```bash
curl https://api.recurso.dev/v1/customers \
  -H "Authorization: Bearer $RECURSO_API_KEY"
```

Mutating requests also accept an `X-Idempotency-Key` header so retries never
duplicate work — see [Idempotency](/advanced/idempotency).

## Revoke a Key

If a key is compromised:

1. Go to **Settings → API Keys**
2. Click **Revoke** on the compromised key
3. Generate a new key
4. Update your applications

<Info>
Revoking a key is immediate and permanent.
</Info>
