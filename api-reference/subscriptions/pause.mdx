---
title: "Pause Subscription"
openapi: "POST /v1/subscriptions/{id}/pause"
description: "Pause an active subscription"
---

## Path Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `id` | string | The subscription ID |

## Request Body

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `resume_at` | string | No | Scheduled resume date (ISO 8601). If omitted, the subscription stays paused until manually resumed. |

## Example Request

```bash
curl -X POST https://billing.example.com/v1/subscriptions/sub_xyz789/pause \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "resume_at": "2024-05-01T00:00:00Z"
  }'
```

## Response

```json
{
  "id": "sub_xyz789",
  "object": "subscription",
  "customer_id": "cust_abc123",
  "plan_id": "plan_pro_01",
  "status": "paused",
  "quantity": 1,
  "current_period_start": "2024-03-15T00:00:00Z",
  "current_period_end": "2024-04-15T00:00:00Z",
  "paused_at": "2024-03-20T09:00:00Z",
  "resume_at": "2024-05-01T00:00:00Z",
  "cancel_at_period_end": false,
  "payment_gateway": "razorpay",
  "metadata": {},
  "created_at": "2024-03-01T10:30:00Z",
  "updated_at": "2024-03-20T09:00:00Z"
}
```

<Note>
While paused, no invoices are generated and the customer is not charged. The subscription resumes automatically at `resume_at` if provided.
</Note>
