---
title: "Generate Referral Code"
openapi: "POST /v1/referrals/generate-code"
description: "Generate a unique, shareable referral code for a customer."
---

## Body Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `customer_id` | string (UUID) | Yes | The unique identifier of the customer who will own this referral code. |

## Example Request

```bash
curl -X POST https://billing.example.com/v1/referrals/generate-code \
  -H "Authorization: Bearer rsk_live_7f3a9b2c1d4e8f0a" \
  -H "Content-Type: application/json" \
  -d '{
    "customer_id": "d4e5f6a7-b8c9-4d0e-a1f2-b3c4d5e6f7a8"
  }'
```

## Response

```json
{
  "data": {
    "code": "RECURSO-D4E5F6",
    "customer_id": "d4e5f6a7-b8c9-4d0e-a1f2-b3c4d5e6f7a8",
    "url": "https://recurso.dev/r/RECURSO-D4E5F6",
    "created_at": "2026-06-23T17:15:00Z",
    "expires_at": null
  }
}
```

## Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `code` | string | The generated referral code. Alphanumeric, uppercase, prefixed with `RECURSO-`. |
| `customer_id` | string | The customer who owns this referral code. |
| `url` | string | A shareable URL that embeds the referral code. New customers who sign up via this link are automatically associated as referred. |
| `created_at` | string | ISO 8601 timestamp of when the code was generated. |
| `expires_at` | string or null | ISO 8601 expiration timestamp, or `null` if the code does not expire. |

## Notes

- Each customer can have one active referral code at a time. Calling this endpoint again for the same customer returns the existing code.
- The referral code is case-insensitive when redeemed but is returned in uppercase for consistency.
- When a new customer signs up using the referral URL, a referral record is automatically created linking the referrer to the new customer.
- To invalidate a referral code, contact support or use the dashboard.

<Tip>
Embed the `url` field directly into email campaigns or share buttons to make referrals seamless for your customers.
</Tip>
