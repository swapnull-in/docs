---
title: "List Referrals"
openapi: "GET /v1/referrals"
description: "Retrieve a paginated list of all referrals in your organization."
---

## Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `limit` | integer | No | Maximum number of referrals to return. Defaults to `10`, max `100`. |
| `offset` | integer | No | Number of records to skip for pagination. Defaults to `0`. |
| `status` | string | No | Filter by referral status. One of: `pending`, `completed`, `expired`, `cancelled`. |
| `referrer_id` | string | No | Filter referrals by the referrer's customer ID. |

## Example Request

```bash
curl -X GET "https://billing.example.com/v1/referrals?limit=5&offset=0&status=completed" \
  -H "Authorization: Bearer rsk_live_7f3a9b2c1d4e8f0a"
```

## Notes

- Results are ordered by `created_at` in descending order (newest first).
- Use the `referrer_id` filter to build a referral dashboard for a specific customer.
- Referrals with status `expired` are those where the referred customer did not complete a payment within the referral window (default 30 days).
- The `meta` object provides pagination details to help you iterate through all results.

<Tip>
Combine `status=pending` with `referrer_id` to show a customer how many of their referrals are awaiting completion.
</Tip>
