---
title: "Get Dunning Weights"
openapi: "GET /v1/analytics/dunning/weights"
description: "Retrieve the learned weights that Recurso's adaptive dunning engine assigns to each action."
---

## Query Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `context_key` | string | No | Filter weights by a specific context key (e.g., `plan:pro`, `country:US`, `failure_reason:insufficient_funds`). Returns all weights if omitted. |
| `limit` | integer | No | Maximum number of weight records to return. Defaults to 50, max 200. |
| `offset` | integer | No | Number of records to skip for pagination. Defaults to 0. |

## Example Request

```bash
curl -X GET "https://billing.example.com/v1/analytics/dunning/weights?context_key=failure_reason:insufficient_funds" \
  -H "Authorization: Bearer rsk_live_7f3a9b2c1d4e8f0a"
```

## Notes

- Weights are updated continuously as the dunning engine processes retry outcomes.
- A higher `average_reward` indicates the action is more effective in the given context.
- The `sample_count` reflects how many times the action has been tried in that context. Higher counts yield more reliable weight estimates.
- Common `context_key` patterns include `plan:<slug>`, `country:<code>`, `failure_reason:<reason>`, and `amount_range:<bucket>`.

<Tip>
Weights with a `sample_count` below 100 may not yet be statistically significant. Allow the engine time to explore before drawing conclusions.
</Tip>
