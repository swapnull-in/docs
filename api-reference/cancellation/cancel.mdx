---
title: "Cancel Subscription"
openapi: "POST /v1/subscriptions/{id}/cancel"
description: "Cancel an active subscription immediately or at the end of the current billing period."
---

## Path Parameters

| Parameter | Type   | Description                              |
|-----------|--------|------------------------------------------|
| `id`      | string | The unique identifier of the subscription (e.g., `sub_3jF6RtKmWp`). |

## Request Body

| Parameter              | Type   | Required | Description                                                                 |
|------------------------|--------|----------|-----------------------------------------------------------------------------|
| `cancel_at_period_end` | bool   | No       | If `true`, the subscription remains active until the current period ends. Defaults to `true`. |
| `immediately`          | bool   | No       | If `true`, cancels the subscription right away and prorates the final invoice. Defaults to `false`. |
| `reason`               | string | No       | A cancellation reason identifier (e.g., `too_expensive`). Use the List Cancellation Reasons endpoint to retrieve valid values. |

When `cancel_at_period_end` is `true` and `immediately` is `false` (the default), the subscription stays active until the current billing period concludes. Setting `immediately` to `true` overrides `cancel_at_period_end`.

## Example Request

```bash
curl -X POST https://api.recurso.io/v1/subscriptions/sub_3jF6RtKmWp/cancel \
  -H "Authorization: Bearer sk_live_26PHem9AhJZvU..." \
  -H "Content-Type: application/json" \
  -d '{
    "cancel_at_period_end": true,
    "reason": "too_expensive"
  }'
```
