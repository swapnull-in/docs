---
title: "Get Portal Data"
openapi: "GET /v1/portal/{tenant_id}/{customer_id}"
description: "Retrieve structured portal data as JSON for building custom portal frontends."
---

## Path Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `tenant_id` | string (UUID) | The tenant ID that owns the customer. |
| `customer_id` | string (UUID) | The ID of the customer to retrieve portal data for. |

## Example Request

```bash
curl https://billing.example.com/v1/portal/ten_d4c3b2a1-0f9e-8765-4321-abcdef012345/cust_a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

## Response

Returns structured JSON with customer details, subscriptions, invoices, and available actions.

```json
{
  "customer": {
    "id": "cust_a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "name": "Acme Corporation",
    "email": "billing@acme.com"
  },
  "subscriptions": [
    {
      "id": "sub_9f8e7d6c5b4a3210",
      "plan_id": "plan_pro_monthly",
      "status": "active",
      "current_period_end": "2026-07-23T00:00:00Z"
    }
  ],
  "invoices": [
    {
      "id": "inv_5d4c3b2a1f0e9876",
      "amount": 249900,
      "currency": "INR",
      "status": "paid",
      "created_at": "2026-06-01T00:00:00Z"
    }
  ],
  "actions": [
    "update_payment_method",
    "cancel_subscription",
    "redeem_gift",
    "download_invoice"
  ]
}
```

<Tip>
This endpoint is designed for custom portal frontends. Use it to build your own
branded customer portal while Recurso handles the subscription and billing logic.
</Tip>
