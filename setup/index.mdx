## What these guides are

Each guide here is a **task-oriented walkthrough**: do this, then this, then verify.
They complement the [concept docs](/concepts) (which explain *what* a feature is)
and the [API reference](/api-reference/introduction) (which documents every
endpoint) — a setup guide is the fastest path from "off" to "working".

Every guide follows the same shape: **Overview → numbered Steps → Verify → Next steps**,
and shows both the **dashboard** path and the **API** where relevant.
