Recurso is a production-grade billing engine you can **actually own**. Plans,
subscriptions, usage, invoicing, and payments sit on a **real double-entry
ledger** — not a table of balances — with **India-first compliance** (GST,
e-invoicing, RBI e-mandates) built in, not bolted on. Self-host it under MIT, or
run it on Recurso Cloud.

## Get started

<Steps>
  <Step title="Clone and run">
    ```bash
    git clone https://github.com/recurso-dev/recurso.git
    cd recurso
    make demo
    ```
    Brings up Postgres and TigerBeetle, the API on `:8080`, and a seeded
    dashboard on `:5173` — with a demo API key printed when it's ready.
  </Step>
  <Step title="Create a customer">
    ```bash
    curl -X POST http://localhost:8080/v1/customers \
      -H "Authorization: Bearer sk_test_12345" \
      -H "Content-Type: application/json" \
      -d '{"name": "Acme Corp", "email": "billing@acme.com"}'
    ```
  </Step>
  <Step title="Start a subscription">
    ```bash
    curl -X POST http://localhost:8080/v1/subscriptions \
      -H "Authorization: Bearer sk_test_12345" \
      -H "Content-Type: application/json" \
      -d '{"customer_id": "cust_123", "plan_id": "plan_pro_monthly"}'
    ```
  </Step>
</Steps>
