Recurso is a production-grade billing engine you can **actually own**. Meter
usage, price it with tiered charges, drain prepaid wallets before cards are
touched, and issue GST-correct e-invoices — all posting balanced debits and
credits to a **real double-entry ledger**, not a table of balances.
Self-host it under MIT, or run it on Recurso Cloud.

## What's in the engine

<Columns cols={2}>
  <Card title="Meter anything, price it your way" icon="gauge-high" href="/advanced/usage-billing">
    Metrics aggregate events by count, sum, max, or unique value. Charges
    price them per-unit, graduated, volume, or package — with sub-paise
    rates computed exactly and billed in arrears on the renewal invoice.
  </Card>
  <Card title="Wallets, commitments & alerts" icon="wallet" href="/advanced/wallets">
    Customers prepay into money wallets that settle invoices before any
    gateway is charged. Contract minimums true-up automatically; usage
    alerts fire once per period before a customer is surprised.
  </Card>
  <Card title="Books that balance" icon="scale-balanced" href="/advanced/ledger">
    Every invoice, payment, refund, and wallet drain posts balanced debits
    and credits to an immutable ledger — with revenue recognition, trial
    balance, and daily reconciliation on top.
  </Card>
  <Card title="India-first, global-ready" icon="earth-asia" href="/compliance/overview">
    Native GST (CGST/SGST/IGST, HSN, Place of Supply), IRP e-invoicing,
    GSTR exports, RBI-compliant UPI AutoPay — plus multi-currency, EU VAT,
    and US sales tax on the same engine.
  </Card>
  <Card title="Recover revenue before it churns" icon="heart-pulse" href="/advanced/growth-overview">
    Dunning campaigns, smart retries, cancel flows, churn scoring, and a
    self-service portal — recovered revenue is attributed and reported.
  </Card>
  <Card title="Developer-first" icon="terminal" href="/api-reference/introduction">
    REST API with OpenAPI 3.1 served from your instance, typed SDKs,
    HMAC-signed webhooks with redelivery, and idempotency keys on every
    money-moving call.
  </Card>

## Get started

<Steps>
  <Step title="Clone and run">
    ```bash
    git clone https://github.com/recurso-dev/recurso.git
    cd recurso
    make demo
    ```
    Brings up Postgres and TigerBeetle, the API on `:8080`, and a seeded
    dashboard on `:5173` — with the demo API key `sk_test_12345` printed
    when it's ready.
  </Step>
  <Step title="Create a customer">
    ```bash
    curl -X POST http://localhost:8080/v1/customers \
      -H "Authorization: Bearer sk_test_12345" \
      -H "Content-Type: application/json" \
      -d '{"name": "Acme Corp", "email": "billing@acme.com"}'
    ```
  </Step>
  <Step title="Start a subscription">
    ```bash
    curl -X POST http://localhost:8080/v1/subscriptions \
      -H "Authorization: Bearer sk_test_12345" \
      -H "Content-Type: application/json" \
      -d '{"customer_id": "cust_123", "plan_id": "plan_pro_monthly"}'
    ```
  </Step>
  <Step title="Record usage">
    ```bash
    curl -X POST http://localhost:8080/v1/usage/events \
      -H "Authorization: Bearer sk_test_12345" \
      -H "Content-Type: application/json" \
      -d '{"subscription_id": "sub_123", "customer_id": "cust_123",
           "dimension": "api_calls", "quantity": 1200}'
    ```
    Usage is aggregated by your billable metrics and rated onto the next
    renewal invoice — [see how rating works](/advanced/usage-billing).
  </Step>
</Steps>
